Contexte du tutoriel¶

Déroulement du tutoriel¶

Tandis que certains utilisateurs pourront faire le tour de ce tutoriel en un jour, ceux qui travaillent chaque exemple dans le détail peuvent probablement compter le finir en une semaine.

Données du tutoriel¶

Le jeu de données utilisé dans ce tutoriel a été récupéré du département U.S. de l’Atlas National Intérieur des Etats Unis. Vous pouvez visiter leur site web sur http://www.nationalatlas.gov. Le jeu de données a été découpé autour de la zone supérieure de la région des grands lacs (Minnesota, Michigan, et Wisconsin) pour réduire la taille de stockage. En plus, des images raster ont été mises à disposition gracieusement par le projet TerraSIP à l’Université du Minnesota. Quand vous utilisez ce tutoriel, vous êtes encouragés à utiliser votre propre jeu de données.

Comme MapServer lui-même, ce tutoriel est ouvert et configurable par n’importe qui. Cela a été fait dans le but que quelqu’un (ou quelques gens) puisse(nt) aider et l’améliorer.

Télécharger les données (et tous les fichiers html) pour ce tutoriel depuis http://download.osgeo.org/mapserver/docs/mapserver-tutorial.zip.

Avant d’utiliser le tutoriel¶

Il y a certains prérequis pour utiliser ce tutoriel:

  1. Les utilisateurs auront besoin d’avoir un serveur web installé et lancé sur leur ordinateur. Ce serveur web doit avoir le support pour les programmes “common gateway interface” (CGI).

  2. Les utilisateurs doivent avoir une compréhension basique des serveurs web et de la sécurité internet. Un serveur web mal configuré peut être facilement attaqué par des personnes mal intentionnés. Au minimum, votre installation logicielle pourra être corrompue et vous perdrez des heures de travail, au pire, votre ordinateur pourra être utilisé pour attaquer d’autres ordinateurs via internet.

  3. Il est recommandé aux utilisateurs de ce tutoriel de lire l’introduction à MapServer avant de passer à ce tutoriel.

  4. Pour utiliser ce tutoriel, les utilisateurs devront avoir un programme CGI MapServer (mapserv ou mapserv.exe) installé sur leur système d’exploitation. Le code source de MapServer est disponible pour le téléchargement en suivant ce lien. La documentation existe sur comment compiler et installer MapServer:

    • pour les utilisateurs UNIX, merci de lire le HOWTO sur l’installation et la compilation de MapServer sous UNIX.

    • Les utilisateurs Windows devraient lire le HOWTO sur l’installation et la compilation de MapServer Win32

    De plus, des binaires précompilés existent pour une nombre important de plateformes (Voir la page de téléchargement.

Problèmes sous Windows et UNIX/Linux¶

Chemins¶

Ce tutoriel a été créé sous Linux/UNIX mais devrait fonctionner avec un minimum de changements sur la plateforme Windows. Les différences principales sont les chemins dans les fichiers .map. Les utilisateurs Windows doivent spécifier la lettre du disque dur où les fichiers du tutoriel sont installés. Voici un exemple:

Un fichier .map UNIX peut inclure éventuellement un paramètre comme ceci:

SHAPEPATH "/data/projects/tutorial/data"

Sous Windows, les mêmes paramètres devraient ressembler à ceci:

SHAPEPATH "C:/data/projects/tutorial/data"

ou:

SHAPEPATH "C:\data\projects\tutorial\data".

Notez que les slashs comme les backslashs fonctionnent sous Windows. Les backslashs habituels fonctionnent peut être bien pour vous si vous voulez faire une distinction entre les liens virtuels (comme les URLs ou les addresses web) et les chemins locaux dans votre fichier .map. Cependant, si vous planifiez de déployer votre application sur UNIX à un moment, vous aurez la tâche ennuyeuse de changer tous les backslashes en slashes.

Tant qu’on est sur le sujet des chemins, gardez à l’esprit que les chemins dans les mapfiles sont typiquement relatifs à la racine du système: le slash (“/”) sous UNIX ou une lettre de lecteur (“C:”) sous Windows. Ceci est vrai excepté quand vous demandez à entrer une URL ou quand vous référencez une URL. Quand vous travaillez avec des fichiers de templates HTML, les chemins sont relatifs au répertoire racine du serveur web. c’est à dire, “/tutorial/” est relatif à “http://demo.mapserver.org/”. Merci de lire http://www.alistapart.com/articles/slashforward/ pour une quelques éclairages sur les URLs.

Exécutables¶

Un autre problème est que les exécutables (ou binaires) ne nécessitent pas une extension .EXE ou .COM, à l’inverse de sous Windows. Si vous utilisez Windows, ajoutez .exe à tous les appels vers “/cgi-bin/mapserv” ou “/cgi-bin/mapserv50” pour qu’ils deviennent “/cgi-bin/mapserv.exe” ou “/cgi-bin/mapserv50.exe”.

Autres ressources¶

D’autres documentations existent pour vous donner une meilleure compréhension des nombreux paramétrages que MapServer propose. Merci de visiter la page de documentation MapServer à l’adresse http://www.mapserver.org. Vous trouverez plusieurs documents HOWTO, pour une première introduction jusqu’à l’usage de MapScript, une interface de scripting pour MapServer.
